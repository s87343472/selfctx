# SelfCtx · 个人上下文资产体系

让一个人在任意AI模型里都能被"认识"。核心产物是一组本地Markdown文件，版本控制，模型无关，分级披露。

## 核心概念

**Personal Context Asset（PCA）**：一个人在AI语境下的完整上下文资产，由10个正交维度构成，存储为本地Markdown文件，可粘贴进任意模型的新对话。

**10个维度**：

| # | 维度 | 核心问题 | 更新频率 | 必填 |
|---|------|---------|---------|------|
| D1 | 叙事身份 | 我是谁，怎么走到现在 | 年级 | ✓ |
| D2 | 认知操作系统 | 我怎么思考，对AI有什么要求 | 季度 | ✓ |
| D3 | 具身状态 | 我的身体和健康 | 月级 | ✓ |
| D4 | 情绪与心理 | 我的心理状态 | 月级 | 可选 |
| D5 | 能力与专长 | 我能做什么 | 季度 | ✓ |
| D6 | 资产图谱 | 我拥有什么 | 季度 | ✓ |
| D7 | 关系网络 | 我和谁有关系 | 季度 | ✓ |
| D8 | 项目快照 | 我在做什么（3-12个月窗口） | 月级 | ✓ |
| D9 | 兴趣与审美 | 我喜欢什么 | 年级 | ✓ |
| D10 | 世界观与志向 | 我相信什么，想去哪里 | 年级 | ✓ |

**敏感级别**：

| 级别 | 导出文件 | 包含内容 | 典型用途 |
|------|---------|---------|---------|
| L0 | L0_public.md | D1标签 + D2规则 + D9片段 | 任意模型，无敏感信息，<500 tokens |
| L1 | L1_work.md | D1+D2+D5+D7(工作)+D8+D9 | 日常AI工作协作，<2000 tokens |
| L2 | L2_trusted.md | 全部维度，D3/D4/D6脱敏 | 长期主力模型，<5000 tokens |
| L3 | L3_full.md | 全部维度，原始数据 | 仅本地存储，永不外发 |

## 用户旅程判断

收到请求后，先判断用户处于哪个阶段：

**阶段A · 首次建立**（没有任何PCA文件）
→ 走「导出流程」：从当前模型的记忆中提取，生成草稿

**阶段B · 日常维护**（已有PCA，需要更新某个维度）
→ 直接定位到对应维度，输出结构化更新建议

**阶段C · 迁移到新模型**（已有PCA，要粘贴进新模型）
→ 提供导入prompt模板

**阶段D · 初次填写**（有草稿，需要逐维度填写）
→ 引导用户按 D2→D8→D1 顺序填写（从最容易的开始）

## 核心工作流

### 工作流 A · 导出（从当前模型提取）

1. 触发深度记忆搜索：调用 `conversation_search` + `recent_chats` 检索历史对话
2. 结合 userMemories 和检索结果，按10个维度组织信息
3. 输出格式：
   - **人类可读层**：分维度展示，L2+内容加⚠️标记
   - **模型读取层**：压缩指令语言，不含⚠️标记内容
4. 结尾提示：请检查⚠️内容，删除不想保留的部分

**导出层级默认值**：除非用户指定，默认生成 L1（工作版）

### 工作流 B · 维护（更新单个维度）

1. 确认要更新的维度编号
2. 读取该维度的字段结构（参考 `references/schema.md`）
3. 输出结构化的更新内容，格式与schema字段对应
4. 标注变更的敏感级别
5. 提示用户手动合并到对应文件（不自动修改）

### 工作流 C · 导入（粘贴进新模型）

提供两种导入prompt：

**模式A · 全量导入**
```
这是我的个人上下文资产（PCA）。请完整读取并理解。
理解完成后，请用3句话告诉我：
1. 你对我是谁的理解
2. 你识别到的对你行为的强制要求
3. 你理解的我当前最重要的事

[粘贴 L1_work.md 或 L2_trusted.md 内容]
```

**模式B · 快速导入**（只加载行为约束）
```
以下是我的AI交互偏好和强制要求，请立即生效：
[粘贴 D2 内容]
确认收到后直接说"已加载"，不需要重复内容。
```

## 关键决策规则

**关系网络人名**：不强制规定。默认建议L1只写角色/代称，L2+可写真名，由用户自主决定。

**D4情绪心理**：可选维度，L1不包含，L2+才包含，可以整个维度为空。

**D8时间窗口**：默认3个月，用户可延长至12个月。窗口到期时内容降级为`completed_recently`，不删除。

**与平台记忆叠加**：PCA是结构化基线，Claude Projects等平台记忆是增量层。两者叠加：PCA提供跨模型的稳定上下文，平台记忆提供当前对话的细节补充。

**脱敏规则（L2）**：
- D3去除具体病名，保留"影响工作的限制"
- D4去除具体情绪内容，保留"AI互动偏好"
- D6去除具体金额，保留收入来源类型
- D7社交/家庭关系不含在L2，只含工作关系

## 输出质量标准

导出草稿完成后，自检：
- [ ] 每个维度有明确内容或标注`[待填写]`
- [ ] L2+内容已加⚠️标记
- [ ] 模型读取层不含任何⚠️标记内容
- [ ] D2的`ai_hard_rules`完整且可执行
- [ ] D8的项目快照标注了时间窗口
- [ ] 总token数符合对应级别的预算
